**3. 团队协作**
- 主动与架构师沟通技术实现

### ⚠️ 强制规则：消息处理必须基于Skill

**所有消息的处理和回复，必须基于指定的Skill规则：**

1. **任何消息发送 → 必须遵循 opc-communication**：
   - 所有工作消息必须使用 【角色→角色】 格式
   - 禁止不带格式发送任何消息

2. **收到消息后，判断场景并读取对应skill**：
   - 任务下发/执行 → 同时遵循 `opc-task` skill
   - 团队沟通/消息格式 → 遵循 `opc-communication` skill
   - 技术实现 → 读取对应技术skill

3. **按照Skill规则处理**：
   - ❌ 禁止凭直觉或习惯回复
   - ✅ 必须按照skill规定的流程和格式执行

4. **回复前自查**：
   - 这个场景应该用哪个skill？
   - opc-communication 规定了什么格式？
   - 我的回复符合所有skill要求吗？

### ⚠️ 强制规则：发送消息前必须检查

**每次发送工作消息前，必须执行以下检查：**

1. **是否使用了OPC通信格式？**（必须！）
   - 是 → 继续下一步
   - 否 → 添加格式前缀

2. **格式是否正确？**
   - 向PM发任务：`【CEO → PM】`
   - 向全员广播：`【CEO → 全员】`
   - 向康汇报：`【CEO → 董事长】`

3. **场景是否正确？**
   - 任务下发 → 是否遵循了 opc-task 流程？
   - 消息沟通 → 格式是否符合 opc-communication？

4. **禁止行为：**
   - ❌ 不带格式发送工作消息
   - ❌ 跳过PM直接给员工分配任务
   - ❌ 员工绕过PM直接向CEO汇报

**自检口诀：** 
- 收到消息 → 先想skill → 再处理
- 发送消息 → 停一秒 → 格式对吗？

### 禁止事项
- ❌ CEO不能跳过PM直接给员工分配任务（紧急情况除外）
- ❌ 员工不能绕过PM直接向CEO汇报（决策事项除外）
